## 1. Purpose

This prompt instructs an AI research agent to produce a comprehensive, persona-by-persona catalog of **the top 10 musical album concepts** for each of the 13 Inspire Family personas, where **every album is engineered to a 12-song standard and every song meets rigorous scoring thresholds**. This execution:

1. Generates the research output for each persona (sentiment, contextual, hit engineering, viral pathway analysis)
2. Creates the 10 album concepts per persona with full metadata and scoring
3. **Deploys the results into the standard Jubilee file system** at `/artifacts/music/albums/` with persona folders, album subfolders, and per-persona `~index.md` catalog files
4. **Does NOT create the per-album Blueprint files** — those are produced in a separate downstream execution phase

The system must use **machine learning, sentiment analysis, contextual analysis, hit engineering analysis, and viral pathway analysis** to ensure every album concept meets every quality standard specified in this document.

## 14. Lyrical Quality Testing Standards

### 14.1 The No Forced Rhyme Rule (Critical)
Rhyme serves meaning, never the other way around. If a line exists only to complete a rhyme and does not advance image, narrative, emotion, or theology, it fails.

**Test:** Would this lyric exist if there were no rhyme requirement?

**Common failures:** Rhyming "grace" with "face" when no facial imagery belongs; inverted syntax purely for rhyme; "oh/yeah" rhyme filler; completing rhymes with meaningless abstraction.

**Preferred alternatives:** Slant rhyme, internal rhyme, deliberate non-rhyme, repetition-based chorus.

### 14.2 Comprehensibility Test
Target audience understands on first hearing.

### 14.3 Relatability Test
Audience can locate themselves in the lyric.

### 14.4 Emotional Compelling Test
Clear emotional response produced (hope, conviction, surrender, joy, lament, awe, repentance, defiance, tenderness, worship, courage, grief, breakthrough).

### 14.5 Faith-Audience Coherence Test
Theological framing matches audience's theological dialect.

### 14.6 Universal Through Specific
Concrete detail evokes universal feeling.

### 14.7 Image Density
Minimum 2 concrete sense-based images per verse.

### 14.8 Originality Test — Banned Clichés
"I'm coming alive," "Break every chain," "Oceans deep," "Here I am," "All my life," "Set a fire," "Holy ground," "My soul cries out," "In the valley," "On the mountain," "Give me more," "Waymaker / miracle worker" — banned unless radically recontextualized.

### 14.9 Prosody Discipline
Word stress matches melodic stress.

### 14.10 Singalong Test
Chorus invites strangers to sing on first hearing.

### 14.11 Theological Precision Per Square Inch
Every line defensible scripturally.

### 14.12 Second-Verse Discipline
Verse 2 advances narrative, not restate verse 1.

### 14.13 First-Hearing Clarity
No footnotes required to follow emotional/theological thread.

## 18. Standard File System Architecture for Music Albums

The Jubilee music catalog is a living file system. Every album concept, blueprint, and production asset lives at a deterministic path so that Jubilee Intelligence, Inspire Family persona runtimes, and the InspireCortex Audio pipeline can find, reference, and update content programmatically.

### 18.1 Root Path
All music album artifacts live under:
```
/artifacts/music/albums/
```

### 18.2 Persona Subfolder Structure
Under the root, one subfolder per Inspire persona using the lowercase `inspire.[firstname]` convention:

```
/artifacts/music/albums/
├── inspire.jubilee/
├── inspire.melody/
├── inspire.zariah/
├── inspire.elias/
├── inspire.eliana/
├── inspire.caleb/
├── inspire.imani/
├── inspire.zev/
├── inspire.amir/
├── inspire.nova/
├── inspire.santiago/
├── inspire.tahoma/
└── inspire.gabriel/
```

Folder names are **lowercase, period-separated, and exact** — `inspire.jubilee`, never `Inspire.Jubilee` or `inspire-jubilee`. The period separator is the locked Jubilee convention.

### 18.3 Per-Persona Folder Contents
Each persona folder contains:

```
inspire.[persona]/
├── ~index.md                               ← Album catalog index (this execution creates)
├── [Album Title 1]/                        ← Album subfolder (this execution creates, empty)
├── [Album Title 2] (OHI)/                  ← OHI album subfolder (this execution creates, empty)
├── [Album Title 3]/
└── ... (10 total album subfolders)
```

### 18.4 Album Folder Naming Convention

**Standard Language albums:**
```
/[Album Title]/
```
Example: `/The King Returns/`

**OHI (Original Hebrew Intent) albums:**
```
/[Album Title] (OHI)/
```
Example: `/Celebrate Yeshua (OHI)/`

**Rules:**
- Album folders use the exact title as written, preserving capitalization and spacing
- OHI albums are flagged with a trailing ` (OHI)` suffix — single space before the parenthesis
- Album folder names must not contain characters that break cross-platform file systems: no `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`. Replace with a dash or remove.
- If an album title contains a colon (e.g., `Fire and Thunder: Volume I`), replace the colon with a dash (e.g., `/Fire and Thunder - Volume I/`)

### 18.5 Per-Album Folder Contents (Future Phase — Not This Execution)
In a later execution phase, each album folder will be populated with its Blueprint file:

```
/[Album Title]/
└── Blueprint - [Album Title].md           ← Created in the Blueprint phase, NOT this execution
```

The Blueprint file contains all 12 songs with full lyrics, music type, per-song scoring, production notes, and complete metadata. **This execution does NOT create Blueprint files.** The album folders are created as empty placeholders to establish the file system structure and reserve the namespace.

### 18.6 Tilde-Prefix Index Convention
The `~index.md` file uses a tilde prefix to ensure it sorts to the top of directory listings alphabetically (tilde sorts before alphanumeric in most file systems and browsers). This guarantees the index is always the first visible file when the persona folder is opened.

### 18.7 This-Execution Scope Boundary (Critical)
This execution creates **only**:
- Missing persona subfolders under `/artifacts/music/albums/`
- Empty album subfolders within each persona folder
- The `~index.md` file in each persona folder

This execution does **not** create:
- Blueprint files (`Blueprint - [Album Title].md`)
- Song lyric files
- Audio files
- Production assets
- Any content inside album subfolders

The album folders remain intentionally empty until the downstream Blueprint execution phase reads the `~index.md` and generates the per-album Blueprint files.

### 18.8 Idempotency
This execution must be idempotent — safe to re-run without duplicating or corrupting structure:
- If a persona folder exists, use it (do not recreate)
- If an album folder exists, use it (do not recreate or empty it)
- If `~index.md` exists, **update it** (do not destroy existing status information — merge updates, preserve non-"Blueprint Pending" statuses for albums whose blueprints have already been created or approved in a later phase)
